# Prompts usados no desenvolvimento do PREVIAGRO

## Geração de JWT

*Prompt usado*
> "Como implementar autenticação JWT em Flask com Flask-JWT-Extended e proteger rotas de CRUD?"

*Objetivo*
> Implementar login, refresh token e proteção de endpoints de criação/edição/exclusão.

*Resultado esperado*
> Rotas de autenticação funcionando com tokens JWT e endpoints protegidos por `@jwt_required()`.

## Criação de CRUD

*Prompt usado*
> "Como organizar um CRUD de produtores em Flask usando SQLAlchemy e arquitetura de service/repository?"

*Objetivo*
> Separar lógica de negócio, persistência e rotas para manter o backend modular.

*Resultado esperado*
> CRUD de produtores funcional com validações e persistência SQLite.

## Arquitetura modular

*Prompt usado*
> "Como estruturar um aplicativo Flask com blueprints, services e repositories para um projeto acadêmico?"

*Objetivo*
> Definir uma arquitetura limpa e escalável para o backend.

*Resultado esperado*
> Estrutura de pastas clara e responsabilidades bem separadas.

## Testes automatizados

*Prompt usado*
> "Quais testes pytest criar para validar autenticação e operações CRUD em uma API Flask?"

*Objetivo*
> Garantir a estabilidade das rotas e do fluxo de autenticação.

*Resultado esperado*
> Casos de teste para login, tokens JWT, criação, consulta, atualização e remoção.

## Refatoração

*Prompt usado*
> "Como refatorar um projeto Flask para remover código legado e melhorar a consistência do JWT?"

*Objetivo*
> Eliminar helpers obsoletos e manter apenas a implementação atual com Flask-JWT-Extended.

*Resultado esperado*
> Código mais limpo, sem duplicação de lógica JWT, e com tratamento de erros consistente.

## Documentação

*Prompt usado*
> "Como escrever README e documentação técnica para entrega acadêmica de um projeto Flask?"

*Objetivo*
> Produzir documentação profissional e completa para avaliação.

*Resultado esperado*
> README, PRD, prompts e diagramas em markdown, prontos para entrega.
